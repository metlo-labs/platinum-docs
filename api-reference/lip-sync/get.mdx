---
title: "Get Lip Sync Inference"
api: "GET https://app.fluxon.ai/api/v1/lipsync/inference/{:id}"
description: "This endpoint gets the lip-sync inference status and response"
---

### Path

<ParamField path="id" type="uuid">
  This is the id of the lip-sync task to fetch
</ParamField>

### Header

<ParamField header="Authorization" type="string">
  This is the user auth token generated for app.fluxon.ai.
</ParamField>

### Response

<ResponseField name="status" type="string">
  The status of the current task. Can be "success", "failure", "started".
</ResponseField>

<ResponseField name="error" type="string">
  The error (if any) encountered by the task. Only present if status was "failure".
</ResponseField>

<ResponseField name="url" type="URL">
  The url (if any) for the lip-synced video generated by the task. Only present if status was "success".
</ResponseField>

<RequestExample>

```bash Example Request
curl --location 'https://app.fluxon.ai/api/v1/lipsync/inference/fda3de6f-6b8f-42d6-a46a-8fdf772bcf19' \
--header 'Authorization: metlo.abcdEFGH12345678zyxw' \
```

</RequestExample>

<ResponseExample>

```json Task Started
{
    "status": "started",
    "error": null,
    "url": null
}
```

```json Task Success
{
    "status": "success",
    "error": null,
    "url": "https://fluxon-bucket.s3.amazonaws.com/00000000-0000-0000-0000-000000000000/00000000-0000-0000-0000-000000000000/output_video.mp4"
}
```

```json Task Failure
{
    "status": "failure",
    "error": "Fluxon encountered an error trying to find the task. Please ensure that the provided task id is correct",
    "url": null
}
```

</ResponseExample>
